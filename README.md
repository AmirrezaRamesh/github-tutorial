# آموزش استفاده از git و github 

گیت(git): یک نرم‌افزار است که برای کنترل نسخه استفاده می‌شود. یعنی به شما کمک می‌کند تغییرات کدهای خود را ذخیره کنید و بتوانید به نسخه‌های قبلی برگردید. مثل یک دفترچه یادداشت برای تغییرات کد.

گیت‌هاب(github): یک وب‌سایت است که به شما اجازه می‌دهد کدهایی که با گیت مدیریت می‌کنید را آنلاین ذخیره کنید و با دیگران به اشتراک بگذارید. مثل یک شبکه اجتماعی برای برنامه‌نویس‌ها که می‌توانند با هم روی پروژه‌ها کار کنند.

## خلاصه-نکات
خلاصه نکات مهم در این قسمت آمده است. مراحل را بخوانید و اگر در مرحله ای با سوالی مواجه شدید ، میتوانید به همان قسمت در گزارش بروید و جزئیات مرحله را به طور کامل مطالعه کنید
1. گیت و گیت هاب دسکتاپ را نصب کنید(در صورت استفاده از linux به قسمت های 5 ، 6 ، 7 و 8 گزارش مراجعه کنید)
2. در برنامه گیت هاب دسکتاپ repository پروژه را در مسیر معینی clone کنید.
3. در گیت هاب دسکتاپ به branch مربوطه بروید(بالای صفحه).
4. فایل ها و کد های مد نظر خود را در فولدر clone شده آپلود یا ویرایش کنید. دقت کنید گیت هاب دسکتاپ باید تغییرات را بشناسد.
5. در زمان commit و push کردن تغییرات مطمئن شوید در پیام خود ، توضیحات مناسب را ذکر می کنید.
6. هر بار قبل از شروع به کار گزینه fetch origin را در گیت هاب دسکتاپ انجام دهید تا مطمئن شوید با آخرین نسخه branch خود آپدیت هستید.
7. سعی کنید به منظم بودن branch ها اهمیت بدهید تا تحلیل و بررسی و گاها استفاده از فایل ها و کد های شما برای بقیه ممکن باشد
8. با تاکید مدیر پروژه حتما در کنار کد و فایل های خود ، README.md را به روز کنید. مستند سازی در ادامه پروژه حائز اهمیت است ( قسمت 9 گزارش برای آموزش README)

## نصب-بر-روی-ویندوز 
مراحل برای نصب روی ویندوز به صورت زیر است:
1. در مرورگر خود git را سرچ کرده و نسخه ویندوز را از سایت رسمی گیت دانلود کنید.
2. فایل دانلود شده را نصب و گزینه های پیش فرض را موقع نصب قبول کنید ( در صورت نیاز بعدا قابل تغییر می باشد) و نصب به راحتی انجام می شود.
3. در مرورگر خود github desktop را سرچ کرده و از سایت رسمی گیت هاب آن را دانلود کنید و برنامه را نصب کنید.
4. صفحه cmd را باز کنید و دستور `git` را وارد کنید. در صورت مشاهده جزئیات در مورد این دستور ، یعنی git به درستی بر روی سیستم شما نصب شده است.
   
- **نکته: نرم افزار گیت یک سری command دارد که برای کنترل پروژه ضروری هستند. اما در نسخه ویندوز میتوان با نصب github desktop به جای استفاده از command line ، از نسخه گرافیکی استفاده کرد که کار ساده تر میشود. اما در هر صورت باید با اسم و کاربرد command ها آشنا بود که در همین قسمت هنگام توضیح github desktop بهش پرداخته می شود. برای نحوه کار با command line به قسمت لینوکس که پایینتر توضیح داده شده مراجعه کنید.**


   ## مفاهیم-اولیه-گیت
    - repository : یک پوشه که تمام کدها و تاریخچه تغییرات را ذخیره می‌کند. مثل یک بایگانی که همه فایل‌ها و نسخه‌های قبلی آن‌ها در آن نگهداری می‌شود. معمولا بر روی فضای ابری گیت هاب ساخته میشود و تغییرات به صورت آنلاین ذخیره و ثبت میشوند.
      
    - clone : دستوری برای کپی کردن کامل یه repository از گیت هاب بر روی سیستم شخصی خود. بعد از استفاده از این دستور ، پوشه قرار گرفته بر روی رایانه شخصی، به گیت هاب لینک است و می توان تغییرات را دریافت و همچنین اعمال کرد
      
    - commit : خیره کردن تغییرات در مخزن. هر کامیت مثل یک عکس از وضعیت فایل‌ها در یک زمان مشخص است. به این ترتیب می‌توان بعدها به این نقطه بازگشت.
      
    - push : فرستادن تغییرات (کامیت‌ها) از کامپیوتر به گیت‌هاب یا یک مخزن آنلاین. یعنی آپلود کردن تغییرات برای به اشتراک‌گذاری.
      
    - pull : دریافت تغییرات جدید از مخزن آنلاین به کامپیوتر. یعنی دانلود کردن آخرین تغییرات برای همگام‌سازی.
      
    - branch : .یک مسیر مستقل برای ایجاد تغییرات بدون آسیب زدن به کد اصلی. می‌توان تغییرات را روی یک شاخه آزمایش کرد و بعد آن را با شاخه اصلی ترکیب کرد.
      
  - **نکته:  تفاوت بین commit و push این است که دستور commit تغییرات شما را صرفا بر روی سیستم شما ذخیره میکند و به کمک git یا بر روی vscode( به کمک extension گیت) ، سابقه قابل مشاهده است اما با استفاده از دستور push تغییر مد نظر ، بر روی گیت هاب به اشتراک گذاشته می شود. همچین مفهوم مشابهی برای fetch و pull وجود دارد که دستور اولی ، تغییرات را از وبسایت دریافت و به شما نشان میدهد ، و دستور دومی علاوه بر اینکار ، فایل های جدید را جایگزین فایل های قبلی میکند**


## ساخت-ویرایش-کلون-repository
    
- در وبسایت گیت هاب یک اکانت بسازید و وارد شوید. در صفحه اصلی ، سمت چپ لیست repo ها قابل دیدن می باشد. همانجا یک گزینه new برای ساخت رپو جدید وجود دارن که باید روی آن کلیک کنید.
- یک اسم برای repo انتخاب کنید ، پرایوت یا پابلیک بودن را تعیین کرده و تیک add README.md را بزنید و repo را بسازید.
- به غیر از ساخت repo ، میتوانید در repo بقیه افراد نیز عضو شوید و با همکاری هم پروژه ها را پیش ببرید. لینک دعوت به ایمیل شما و inbox های گیت هاب ارسال می شود.
- حال باید نرم افزار github desktop را اجرا کنیم. روی گزینه لاگین بزنید و از طریق مرورگر باز شده وارد اکانت گیت هاب خود بشوید و آن را با برنامه لینک کنید.
- در برنامه بعد از لاگین ، لیست repo هایی که ساخته اید یا در آنها عضو هستید باز می شود. repo پروژه ویلچر هوشمند را انتخاب کرده و آنرا در **یک مسیر معین** clone کنید. حال شما repo پروژه را بر روی رایانه شخصی خود دارید.
- حال بدون نیاز به command line میتوانید تغییرات را pull, commit و push کنید.
- در پوشه clone شده پروژه ، هر فایلی قرار بدهید یا هر فولدری بسازید ، github desktop تغییرات را شناسایی و در داخل برنامه برای شما نمایش می دهد. شما بعد از نمایش داده شدن ، میتوانید گزینه commit (پایین سمت چپ) برای ذخیره تغییرات و push(بالا سمت راست) برای ارسال تغییرات بر روی گیت هاب ، انتخاب کنید. همچنین بالای صفحه گزینه fetch origin است که نشان میدهد آخرین بار چه زمانی تغییرات وبسایت با فایل محلی شما تطتابق داده شده و اگر تغییری دیده شود ، برای pull کردن به شما اطلاع می دهد.

## نظم-پروژه-به-وسیله-branch
در گیت هاب repo ها به صورت پیش فرض شاخه اصلی به اسم main دارند. برای سازماندهی بیشتر ، امکان ساخت شاخه های بیشتر وجود دارد. فلسفه اصلی وجود شاخه های بیشتر ،‌ برای انجام تست و اعمال تغییرات بر روی فایل های اصلی در فضایی جداگانه( بدون تغییر مستقیم بر روی فایل) شکل گرفته است. مخصوصا هنگام کار چند توسعه دهنده، بر روی یک موضوع است. شخصی در حال کار بر روی کد control.cpp است. او میخواهد به صورت جداگانه چند تغییر بر روی کد اعمال و تغییرات را تست کنید و از آنجایی که فایل اصلی قرار گرفته بر روی شاخه اصلی را نمیخواهد تغییر دهد ، شاخه ای با اسم "test" می سازد و آنرا به شاخه " main " سورس می کند. حال تمام فایل های شاخه اصلی بر روی شاخه فرعی test کپی می شود و فرد میتواند تغییرات مد نظر را اعمال کند و در نهایت در صورت تمایل می تواند تغییرات و فایل های جدید را بر روی شاخه اصلی ، merge کند. بدیهی است که نباید کد ها و فایل های شاخه اصلی نامنظم و ناخوانا باشند. به همین علت ، میتوان در شاخه های متفاوت ، توسعه قسمت های مختلف پروژه را پیش برد و در نهایت آنها را بر روی شاخه اصلی اضافه کرد.

  ## نصب-و-اجرا-بر-روی-ترمینال-linux

- `git config --global user.name "username"`

  ## ساخت-PAT-امنیتی

برای clone کردن ، باید اکانت خود را برای git معرفی کنیم که در سیستم عامل لینوکس از طریق username و password ممکن نیست و نیاز به personal access token داریم. به وسیله این token میتوانیم با سیستم های مختلف لینوکسی ، هنگام استفاده از دستورات مهم git مثل clone و push و pull ممکن است به این token نیاز داشته باشیم. در صفحه اصلی github ، بالا سمت راست بر روی پروفایل خود کلیک کرده ، پایین پنل باز شده گزینه setting را انتخاب کنید. در صفحه تنظیمات ،‌ نوار سمت چپ را به پایین بروید و بر روی developer setting کلیک کنید. در صفحه جدید سمت چپ ، گزینه personal access token را انتخاب کرده و روی PAT(classic) کلیک کنید. در وسط صفحه بر روی generate new token بزنید. یک اسم دلخواه برای token انتخاب کرده ، کمی پایین تر تمام دسترسی های لازم(به صورت باکس هایی هستند که باید تیک زده شوند) را برای token در نظر بگیرید که هر زمان با استفاده از این token گیت شمارا شناسایی کرد ،‌ دسترسی به انجام تمام کار های لازم را داشته باشید. token را بسازید و آنرا حتما در جایی امن یادداشت کنید زیرا دیگر نمایش داده نمی شود. در مرحله بعد به کمک PAT ، میتوانیم repo را clone کنیم.

  ## اضافه-کردن-repo-به-سیستم-شخصی-با-دستور-clone 

 1.در وب گیت هاب وارد repo شوید و در صفحه اصلی بمانید. از بالای مرورگر مسیر را کپی کنید. باید به فرمت `https://github.com/user/repository` باشد. 
 2.حال که مسیر repo را کپی کرده اید ،‌ ترمینال لینوکس را باز کرده و به کمک دستور `cd` به directory دلخواه خود بروید.
 3.سپس دستور زیر را وارد کنید. دقت کنید `<username>` را با یوزرنیم اصلی خود ، `<PAT>` را با  PAT جنریت شده و username و repo هم همانطور که گفته شد جایگزین کنید.به مثال زیر دقت کنید

 - `git clone https://<username>:<PAT>@github.com/user/repository `
  
 - `git clone https://amir:ghp_fSFgfwsKOdVVYV1vBrOANtYDeI3iz5ME@github.com/SheikhJafary/arsheida-smart-wheelchair`

4.بعد از clone کردن repo به طور کلی باید بتوان از دستورات گیت در مسیری که repo را قرار داده ایم از دستورات گیت استفاده کنیم. اما در شرایطی ممکن است مجددا نیاز به authentication داشته باشیم و مجددا PAT خود را باید به git معرفی کنیم. بعد از استفاده از دستور بالا برای clone کردن ، میتوان از دستور زیر برای ذخیره کردن مشخصات authentication استفاده کرد که مجددا نیاز به استفاده از PAT در دستورات نباشد. به طور کلی اگه در command های قبلی از PAT استفاده کرده باشید مستقیما باید دستور زیر اعمال شود اما ممکن است بعد از این دستور ، git از شما یوزرنیم و پسوورد(که همان PAT) است را بخواهد که باید آن ها را وارد کنید و بعد ذخیره می شوند.
   - `git config --global credential.helper store`
     
    
   
  ## لیست-command-های-مورد-نیاز
  
  ```bash
git status # to see the changes made in the repo

git add file # add a file or a modified file to git ready to be commited

git add . # add all the files(including modified ones) to git ready to be commited

git commit -m "commit message" # to commit changes with a message

git push origin <branch-name> #push the changes to the branch we want e. g. main or electronic

#for example
git add code.py
git commit -m "new python code uoploaded"
git push origin control

git pull origin <branch-name> # to download the latest version of the files of a branch to your PC

git pull origin AI # download new & modified files for the AI branch and replace them with the files you have on your PC

git checkout <branch-name> #switch from one branch to another

git checkout mechanic #here we switch from main branch to mechanic branch. now the files of mechanic branch is accessible

git branch #list branches that we have locally siwtched to, meaning at first it lists only main branch but after checkout commands, it shows other branches too

git checkout -- file.name # go back to last commit of the modified file. it could be a python code or even a stl mesh file

git log #to see history of commits

```

   ## گزینه-پروژه-در-گیت-هاب

   ## سازگاری-git-و-vscode
در صورت نصب بودن git بر روی سیستم شما ، در هنگام استفاده از VScode ، این برنامه از git پشتیبانی می کند. به این معنا که میتوانید در repo ای که روی سیستم clone شده ، با کمک vscode کد بزنید و مستقیما در فضای برنامه vscode ، تغییرات کد را commit و push کنید.
برای انجام این امر ، بالای صفحه روی file بزنید ، سپس گزینه open folder را انتخاب کنید و فولدر clone شده از github را انتخاب کنید. سپس نوار سمت چپ را مشاهده کنید و لوگوی git (به صورت چند نقطه که به وسیله خط به هم متصل هستند) قابل رویت است. تصویر زیر ، کلیت پنل git در vscode را نشان می دهد.

## ستاپ جدید پروژه به وسیله sub module های گیت هاب
قبلا قرار بود هر رول تیم ، به صورت یک شاخه سورس شده به main باشد. مشکل محدود کردن دسترسی و نیاز به clone کردن فایل های بقیه رول ها حتی به صورت عدم نیاز به آنها هنگام اضافه کردن به سیستم شخصی وجود داشت. با ستاپ جدید ،‌ هر رول repo خود را دارد و افراد آن بخش ، فقط به repo خود دسترسی دارند. حال repo اصلی پروژه نیز ساخته شده است. مالکیت این repo با مدیر پروژه می باشد و بقیه repo ها به عنوان sub module اضافه شده اند. برای اپدیت بودن repo اصلی پروژه بر روی سیستم شخصی ،‌ از دستور pull نباید استفاده کرد و باید submodule ها را به روزرسانی کرد
